# Solo Asset Manager

Solo Asset Manager 是一个用于管理本地游戏图片资源与音频资源的轻量级工具。

工具使用 Python 后端与 Vite 前端运行，不需要打包为桌面应用。

资源文件保存在本地 Windows 文件夹中。
工具通过扫描目录、移动文件、记录来源、维护 JSON 索引的方式管理资源。

---

## 1. 工具概述

本工具用于管理个人游戏项目中的图片资源与音频资源。

支持资源格式：

```text
图片：png / svg
音频：ogg / wav
```

资源整理流程：

```text
待选池 → 当前项目
待选池 → 仓库
```

资源文件本体保存在总资产目录中。
JSON 文件只保存路径、索引、来源、标签和备注等辅助数据。

---

## 2. 总资产目录

工具只需要设置一个总资产目录。

示例：

```text
F:/OwnerAssets
```

总资产目录下包含固定系统目录：

```text
OwnerAssets/
  Project/
  待选池/
  仓库/
```

目录说明：

| 目录 | 说明 |
| --- | --- |
| Project | 项目目录集合 |
| 待选池 | 尚未整理的临时资源区域 |
| 仓库 | 全局长期保留资源库 |

一个总资产目录对应一个资产管理中心。
每个资产管理中心只有一个全局仓库。

---

## 3. 项目目录

项目存放在 `Project` 目录下。

新建项目等于在 `Project` 下创建一个一级目录。

示例：

```text
OwnerAssets/
  Project/
    项目A/
    项目B/
```

每个项目下固定包含以下基础目录：

```text
项目A/
  图片/
  音频/
```

`图片` 与 `音频` 是固定路径行为。
工具创建项目时自动创建这两个目录。

项目切换不是高频操作。
工具以当前项目为主要操作对象。

---

## 4. 分类

分类是前端展示概念。
后端通过文件夹路径管理分类。

项目资源分类位于项目的 `图片` 或 `音频` 目录下。

示例：

```text
OwnerAssets/
  Project/
    项目A/
      图片/
        背景图/
        图标/
        武器/
      音频/
        BGM/
        UI/
        特效/
```

仓库资源分类位于仓库的 `图片` 或 `音频` 目录下。

示例：

```text
OwnerAssets/
  仓库/
    图片/
      图标/
      背景图/
    音频/
      BGM/
      特效/
```

分类规则：

* 分类等于 `图片` 或 `音频` 下的一层子目录
* 新建分类等于创建子目录
* 分类不默认创建
* 前端显示分类名称
* 后端保存和扫描实际文件夹路径

---

## 5. 待选池

待选池用于放置尚未整理的资源。

待选池固定路径：

```text
OwnerAssets/
  待选池/
```

待选池不需要子文件夹。

待选池资源支持以下操作：

* 扫描
* 查看
* 预览
* 试听
* 编辑标签
* 编辑备注
* 记录来源
* 移动到当前项目
* 移动到仓库
* 打开所在文件夹

待选池资源不强制使用编号文件名。
待选池资源在移动到项目或仓库时完成分类归档和自动改名。

---

## 6. 仓库

仓库用于存放未进入当前项目、但具有保留价值的资源。

仓库固定路径：

```text
OwnerAssets/
  仓库/
    图片/
    音频/
```

仓库下的 `图片` 与 `音频` 是固定路径行为。
仓库初始化时自动创建这两个目录。

仓库资源通过 `图片` 或 `音频` 下的子目录进行分类。
仓库不承担废弃区功能。

---

## 7. 资源类型识别

工具根据文件扩展名识别资源类型。

| 扩展名 | 资源类型 |
| --- | --- |
| png | 图片 |
| svg | 图片 |
| ogg | 音频 |
| wav | 音频 |

不符合上述格式的文件不进入资源扫描结果。

---

## 8. 文件命名规则

待选池资源不强制使用编号后缀。

项目资源和仓库资源必须使用编号后缀。

编号后缀格式：

```text
_1
_2
_3
```

示例：

```text
click_1.wav
click_2.wav
icon_sword_1.png
icon_sword_2.png
```

从待选池移动到项目或仓库时，工具自动生成目标文件名。

自动命名规则：

* 去掉原文件名末尾已有的 `_数字` 后缀
* 在整个总资产目录中查找同名资源的最大编号
* 使用下一个编号生成目标文件名
* 用户不需要手动处理同名冲突

示例：

```text
待选池/click.wav
```

如果总资产目录中已存在：

```text
Project/项目A/音频/UI/click_1.wav
仓库/音频/特效/click_2.wav
```

移动后的新文件名为：

```text
click_3.wav
```

---

## 9. 资源移动规则

工具不要求用户手动选择完整目标路径。

目标路径由以下部分组成：

```text
基础目录 + 分类目录 + 自动编号文件名
```

基础目录由工具根据资源类型和目标区域自动确定。

| 目标区域 | 图片基础目录 | 音频基础目录 |
| --- | --- | --- |
| 当前项目 | Project/当前项目/图片/ | Project/当前项目/音频/ |
| 仓库 | 仓库/图片/ | 仓库/音频/ |

用户只需要选择目标区域和分类。

移动到当前项目示例：

```text
待选池/icon_sword.png
→ Project/项目A/图片/武器/icon_sword_1.png
```

移动到仓库示例：

```text
待选池/click.wav
→ 仓库/音频/UI/click_1.wav
```

## 10. 来源记录

资源可以记录来源信息。

来源类型：

| 类型 | 说明 |
| --- | --- |
| 自制 | 自行制作的资源 |
| 网络来源 | 来自网络的资源 |

来源字段：

```text
来源类型
来源链接
备注
```

来源信息保存在 JSON 数据中。

---

## 11. 标签

目录结构用于表达主要用途。
标签用于补充目录无法表达的附加信息。

标签示例：

```text
可商用
需确认版权
临时
可循环
需修改
氛围感强
```

标签规则：

* 一个资源可以绑定多个标签
* 待选池资源可以绑定标签
* 项目资源和仓库资源可以绑定标签
* 标签不影响文件实际路径
* 标签用于搜索与筛选
* 标签信息保存在 JSON 数据中

## 14. 数据保存

工具使用 JSON 保存辅助数据。

保存内容：

```text
总资产目录
当前项目
资源索引
资源来源
标签
备注
界面设置
移动记录
```

资源文件本体保存在 Windows 文件夹中。
JSON 不保存文件内容。

建议数据文件：

```text
settings.json
pending_assets.json
project_assets.json
library_assets.json
tags.json
sources.json
history.json
```

文件说明：

| 文件 | 说明 |
| --- | --- |
| settings.json | 总资产目录、当前项目、扫描规则、界面设置 |
| pending_assets.json | 待选池资源索引 |
| project_assets.json | 项目资源索引 |
| library_assets.json | 仓库资源索引 |
| tags.json | 标签列表、标签颜色、标签说明 |
| sources.json | 来源类型、常用来源网站 |
| history.json | 移动记录、修改记录 |

15. 编辑备注
